## Responsibilities

Maintainers should:

- Review pull requests for accuracy and safety.
- Reject secrets, private data, unsafe payment guidance, and destructive production advice.
- Keep `upload-ready/` files in sync with source Markdown files.
- Keep README, contribution rules, and GPT setup files updated.
- Prefer practical Laravel and NativePHP guidance with commands, code, and verification steps.

## Review Checklist

- Does the contribution solve a real-world Laravel/NativePHP problem?
- Is the advice safe for production?
- Are secrets or private data absent?
- Are destructive commands clearly warned against?
- Are version-specific commands marked clearly?
- Are source files updated, not only upload-ready files?
- Is README updated if a new topic/file was added?
